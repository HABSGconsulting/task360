# 01 — What This Repo Does

> **Read this first.** Everything else in `docs/` builds on this.

---

## The One-Line Version

This repo builds a **Task360 Intelligence Database** — an AI-enriched dataset of 546 funded Indian startups that helps Task360 identify, prioritise, and engage founders as potential clients.

---

## The Problem It Solves

Task360 has a clear service offering (compliance, tax, legal, fundraising advisory) but no structured way to identify *which* of the thousands of funded Indian startups are most likely to need those services right now — and no ready-made engagement material for each one.

This repo solves that by:

1. Taking a clean base database of funded Indian startups (546 rows)
2. Mapping each startup to the Task360 service clusters most relevant to its stage and sector
3. Using Gemini AI to generate per-startup intelligence: scores, work areas, hook lines, proposals, and engagement strategies
4. Producing clean, prioritised outputs that Task360 can use directly

---

## What This Repo Is NOT

- It is **not** an outreach pipeline — no emails, no personal LinkedIn, no contact data
- It is **not** a research database — no macroeconomic analysis, no regression models
- It is **not** part of the internship academic project — that lives in `HABSGconsulting/internship`

This repo is purely an **intelligence and business development tool** for Task360.

---

## The Two-Layer Architecture

### Layer 1: Base Database (`task360_base.json`)

Built from `FUST_cleaned.csv` (sourced from the internship repo). Contains 12 factual fields per startup — no personal contact data. Taxonomy-mapped, funding-normalised, schema-compliant. Built by `scripts/01_build_base_db.py`. Runs without any API calls.

### Layer 2: Intelligence Layer (`task360_intelligence.json`)

AI-generated fields appended to each base record. Includes partnership score, work areas, hook line, one-page proposal, and engagement strategy. Generated by `scripts/03_generate_intelligence.py` using Gemini 3.1 Flash-Lite. Built incrementally — resume-safe.

## The Flow

```
1. SOURCE DATA     HABSGconsulting/internship → data/FUST_cleaned.csv
   (546 funded Indian startups, 17 fields)
         ↓
2. STEP 0          bucket_to_taxonomy_map.json + stage_to_services_map.json
   (static reference files, human-defined once)
         ↓
3. SCRIPT 01       scripts/01_build_base_db.py
   (clean + map + normalise → task360_base.csv + task360_base.json)
         ↓
4. SCRIPT 02       scripts/02_validate_base.py
   (QA check → validation_report.txt)
         ↓
5. SCRIPT 03       scripts/03_generate_intelligence.py
   (Gemini loop → task360_intelligence.json, grows daily)
         ↓
6. SCRIPT 04       scripts/04_export_final.py
   (merge + sort + split → task360_master.json/csv + by_category/)
         ↓
7. FINAL OUTPUT    task360_priority.csv — sorted by partnership_score DESC
   (Task360 uses this for prioritised BD outreach)
```

| File | Purpose |
